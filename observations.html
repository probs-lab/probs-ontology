Observations¶

An Observation represents a single piece of information about the system.

System roles¶

Fig. 2 How the Roles of Observations relate to different parts of the MEFA system structure that can be measured. An observation with role ProcessInput and ProcessOutput refers to a specific single flow in the system, and must be linked to both a process and an object. In contrast, SoldProduction, Consumption, Import and Export refer to the sum over all processes and must be linked only to an object.¶

Concept reference¶

Observation¶

One piece of observed data.

TimePeriod¶

Describes the temporal period over which an Observation is measured. Often it is a year period, but it could be financial years, calendar years, months, days, hours, minutes, seconds, or arbitrary spans.

Region¶

Describes the spatial area over which an Observation is measured.

Role¶

The type of data represented by the Observation.

Specific roles:

Flow¶

Subclass of Role for Observations which measure flows, that is, movement of Objects between Processes.

Import¶

Subclass of Flow which describes an Object entering the Region of interest.

Export¶

Subclass of Flow which describes an Object leaving the Region of interest.

ProcessInput¶

Subclass of Flow which describes an Object entering a specific Process.

ProcessOutput¶

Subclass of Flow which describes an Object leaving a specific Process.

SoldProduction¶

Subclass of Flow which describes the total output of an Object from all Processes that produce it. The term “sold production” is defined by PRODCOM, as opposed to the goods that may have been produced within the factory boundary, but haven’t yet left the factory gate.

Stock¶

Subclass of Role for Observations which measure stocks, that is, the accumulation of Objects within Processes.

Object properties:

objectDefinedBy¶

Links an Observation to the Object it describes

processDefinedBy¶

Links an Observation to the Process it describes, where applicable (some, such as SoldProduction, relate to only an Object independent of Processes).

hasTimePeriod¶

Links an Observation to the TimePeriod in which it was measured.

hasRegion¶

Links an Observation to the Region in which it was measured.

hasRole¶

Links an Observation to the Role representing the type of data which it measures.

metric¶

Describes the way in which a measurement is quantified, e.g. by mass or volume (linked to QUDT QuantityKind)

Data properties:

measurement¶

The numeric value of the Observation.
